This document lists all the features of the website, the technologies used for each, a simple explanation of how they work, and the files where the code is written.

1. Convert to PDF
----------------
- Technology: React (Frontend), Node.js, Express, LibreOffice (Backend).
- Explanation: This feature allows you to upload Word documents, Excel sheets, PowerPoint presentations, and common images (JPG, PNG) to convert them into a professional PDF format.
- Code Files:
  - Frontend: /frontend/src/pages/PdfTools.jsx
  - Backend: /backend/src/services/pdfService.js (Conversion logic)
  - Backend: /backend/src/routes/pdfRoutes.js (API route)

2. Convert from PDF
------------------
- Technology: React (Frontend), Node.js, Express, LibreOffice/PDF-Lib (Backend).
- Explanation: This tool takes an existing PDF and converts it back into editable formats like Microsoft Word (.docx) or Excel (.xlsx), or extracts pages as high-quality images.
- Code Files:
  - Frontend: /frontend/src/pages/PdfTools.jsx
  - Backend: /backend/src/services/pdfService.js
  - Backend: /backend/src/routes/pdfRoutes.js

3. Compress PDF
---------------
- Technology: React (Frontend), Node.js, Express, Ghostscript (Backend).
- Explanation: It reduces the file size of your PDF documents (using levels like "Extreme", "Recommended", or "High Quality") so they are easier to share via email.
- Code Files:
  - Frontend: /frontend/src/pages/PdfTools.jsx
  - Backend: /backend/src/services/pdfService.js
  - Backend: /backend/src/routes/pdfRoutes.js

4. Protect & Unlock PDF
-----------------------
- Technology: React (Frontend), Node.js, Express, PDF-Lib/QPDF (Backend).

  Deploying on a VPS (like DigitalOcean, Linode, or Hostinger) is the best choice for "N.A.I.R Solutions" because you have full control over the system to install the heavy tools you need (LibreOffice, Ghostscript).

Here is the exact roadmap and the checklist of what you will need to do:

1. The "Must-Have" Checklist for your VPS
Before your code can even start, you must install these on your VPS (assuming Ubuntu):

Node.js & NPM: To run the apps.
PM2: A process manager that keeps your server running 24/7 (even if it crashes or the VPS restarts).
MongoDB: Either install it on the VPS or use MongoDB Atlas (Cloud).
Nginx: To act as a "Reverse Proxy" (this allows users to access your site via nairsolutions.com instead of IP:5000).
The Heavy Lift Tools:
bash
sudo apt update
sudo apt install libreoffice ghostscript  # CRITICAL for conversions
2. The Step-by-Step Deployment Roadmap
Step A: Preparation (Do this now)
You need to make sure your code doesn't have "localhost" hardcoded everywhere.

Frontend: Create a .env file for the frontend where VITE_API_URL points to your VPS IP or Domain.
Backend: Ensure process.env.MONGODB_URI and process.env.JWT_SECRET are used.
Step B: Transferring Code
The easiest way is to push your code to GitHub and then git clone it onto your VPS.

Step C: Running the Backend
On your VPS, navigate to the backend folder and run:

bash
npm install
npm install -g pm2
pm2 start src/server.js --name nair-backend
Step D: Building the Frontend
Since the frontend is a React (Vite) app, you don't "run" it like the backend. You "build" it into static files:

bash
cd frontend
npm install
npm run build
This will create a dist folder. You will tell Nginx to serve the files inside this dist folder to your visitors.

Step E: Domain & SSL (HTTPS)
Use Certbot (Let's Encrypt) to get a free SSL certificate so your site has the "lock" icon and is secure for user logins.
